# ドキュメント生成プロンプト

以下のガイドラインに従って `$1` のドキュメントを生成してください：

## 1. コードベースの分析
- プロジェクトの構造と主要機能を理解
- 使用されている技術スタックを確認
- 既存のドキュメントを確認

## 2. ドキュメントの種類を判断
- **README.md** — プロジェクト全体の概要
- **API ドキュメント** — エンドポイントの詳細
- **コードドキュメント** — JSDoc/docstring
- **ガイド・チュートリアル** — 使い方の説明
- **アーキテクチャドキュメント** — 設計・構造の説明

## 3. ドキュメント生成の原則

### 明確さ
- 簡潔に、具体的に記述
- 必ず使用例を含める

### 正確性
- コードの実装と矛盾しない
- 古い情報を含めない

### ユーザビリティ
- 適切な見出しとセクションで整理
- 関連ドキュメントへのリンクを提供

## 4. 生成後の説明
- 生成したドキュメントの一覧と配置場所
- 次のステップ（レビュー、公開等）
- メンテナンス方法
