# セキュリティレビュー

OWASP Top 10 を中心に、セキュリティ脆弱性を体系的に検出・評価します。

## レビュー手順

### 1. 攻撃面の特定

- 外部からの入力ポイントを列挙する（API、フォーム、URL パラメータ等）
- 信頼境界を特定する（クライアント/サーバー、内部/外部）
- 認証・認可が必要な箇所を確認する

### 2. OWASP Top 10 チェックリスト

#### A01: アクセス制御の不備
- 適切な認可チェックが全エンドポイントにあるか
- 水平権限昇格（他ユーザーのリソースへのアクセス）対策
- CORS 設定が適切か

#### A02: 暗号化の失敗
- パスワードが適切にハッシュ化されているか（bcrypt 等）
- 機密データが平文で保存・送信されていないか
- HTTPS が必須化されているか

#### A03: インジェクション
- SQL/NoSQL インジェクション対策（パラメータ化クエリ、ORM）
- コマンドインジェクション対策
- XSS 対策（出力のエスケープ）

#### A04: 安全でない設計
- 入力のバリデーション（ホワイトリスト方式）
- レート制限の実装
- 適切なエラーメッセージ（内部情報の漏洩防止）

#### A05: セキュリティ設定のミス
- デフォルト設定の変更（パスワード、ポート等）
- 不要な機能・エンドポイントの無効化
- セキュリティヘッダーの設定

#### A07: 認証の不備
- 適切なセッション管理
- トークンの有効期限設定
- ブルートフォース対策

#### A08: ソフトウェアとデータの整合性の不備
- 依存関係の脆弱性チェック
- デシリアライズの安全性

#### A09: セキュリティログとモニタリングの不備
- 認証の失敗がログに記録されているか
- 機密情報がログに出力されていないか

### 3. 機密情報管理

- API キー、パスワード、トークンがハードコーディングされていないか
- `.env` ファイルが `.gitignore` に含まれているか
- ログに機密情報が出力されていないか
- エラーメッセージに内部情報が含まれていないか

### 4. 報告書の作成

各脆弱性について以下を記述する:

- **深刻度**: Critical / High / Medium / Low / Info
- **脆弱性の種類**: OWASP カテゴリ
- **影響**: 悪用された場合の影響
- **再現手順**: 脆弱性の再現方法
- **修正方法**: 具体的な修正コード・手順
