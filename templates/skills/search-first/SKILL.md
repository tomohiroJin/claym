# サーチファースト

実装を始める前にコードベースを調査し、既存のパターンや規約を理解します。

## 手順

### 1. 既存実装の調査

- 類似機能が既に実装されていないか検索する
- 再利用可能なユーティリティ・ヘルパーを探す
- 共通コンポーネント・モジュールを確認する

### 2. パターンの把握

- プロジェクトのディレクトリ構造のパターンを確認する
- 命名規則（ファイル名、変数名、関数名）を把握する
- エラーハンドリングのパターンを確認する
- テストの記述パターンを確認する

### 3. 規約・設定の確認

- リンター・フォーマッターの設定を確認する
- コーディング規約ドキュメントを読む
- CI/CD パイプラインの要件を把握する

### 4. 依存関係の確認

- 変更対象が他のモジュールにどう依存しているか
- 変更対象に依存しているモジュールはどれか
- 型定義・インターフェースの契約を確認する

### 5. 調査結果に基づいて実装する

- 既存のパターンに合わせたコードを書く
- 既存のユーティリティを活用する
- プロジェクトの規約に従う
- 不明点があれば確認してから進める

## 調査チェックリスト

- [ ] 類似機能の有無を確認した
- [ ] 再利用可能なコードを探した
- [ ] 命名規則を把握した
- [ ] ディレクトリ構造のパターンを理解した
- [ ] テストパターンを確認した
- [ ] 依存関係を把握した

## 注意事項

- 調査に時間をかけすぎない（目安: 実装時間の10-20%）
- 全てを調べる必要はない。関連する範囲に絞る
- 調査結果は実装計画としてまとめると効率的
