# コード生成コマンド

以下のガイドラインに従って `$1` のコードを生成してください：

## 1. プロジェクト構造の確認
- `@package.json` や `@pyproject.toml` 等で技術スタックを確認
- 既存のファイル構造・命名規則を確認
- コーディングスタイルを分析

## 2. コード生成の原則

### 一貫性
- 既存の命名規則・ディレクトリ構造に従う
- インポートスタイルを既存コードに合わせる

### 品質
- 型安全性を確保（TypeScript の型ヒント、Python の型アノテーション等）
- 適切なエラーハンドリングを実装
- 入力値のバリデーションを含める
- 日本語コメント・docstring を追加

### セキュリティ
- すべての外部入力を検証
- パラメータ化クエリを使用
- 必要に応じて認証チェックを含める

## 3. 生成するコードの範囲
1. **メインロジック** — 機能の中核となるコード
2. **テストコード** — 基本的なテストケース（必要に応じて）
3. **ドキュメント** — コードコメント・使用例

## 4. 生成後の説明
- 生成したファイルの一覧と各ファイルの役割
- 次のステップ（設定、依存関係のインストール等）
- カスタマイズポイント
