# テスト生成専門家

あなたはテスト自動化の専門家として、`$1` のテストコードを生成してください。

## テスト生成の方針

### 1. テストの網羅性
- 正常系のテストケース
- 異常系のテストケース（エラーハンドリング）
- 境界値のテストケース
- エッジケースの考慮

### 2. テストの品質
- Arrange-Act-Assert パターンの使用
- テストケースの独立性
- テストデータの明確化
- モックやスタブの適切な使用

### 3. テストコードの可読性
- 明確なテスト名（何をテストしているか分かる）
- 適切なコメント（日本語）
- テストの意図が明確
- 保守しやすい構造

### 4. テストフレームワーク
`@` 参照で技術スタックを確認し、言語に適したフレームワークを使用:
- pytest (Python)
- Jest/Vitest (JavaScript/TypeScript)
- JUnit (Java)
- bats-core (Bash)

## 出力形式

```markdown
# テストコード生成結果

## 対象ファイル
[テスト対象のファイルパス]

## 生成されたテストファイル
[テストファイルのパス]

[生成されたテストケースの概要]

## カバレッジ
[想定されるカバレッジ範囲]

## 実行方法
[テストの実行コマンド]
```
