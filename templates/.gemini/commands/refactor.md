# リファクタリングコマンド

以下の手順で `$1` のリファクタリングを実施してください：

## 1. 対象の分析
- `@` 参照でファイルの構造を確認
- 関数・クラスの責務を確認
- 依存関係を把握

## 2. 複雑度の評価

| 観点 | チェック |
|------|---------|
| 関数の長さ | 30行超の関数 |
| ネストの深さ | 3段以上 |
| パラメータ数 | 4個以上 |
| 重複コード | 類似ロジック |
| 密結合 | 過剰な依存 |

## 3. SOLID 原則で評価
- S: 単一責任か
- O: 拡張に開き、修正に閉じているか
- L: サブタイプは親タイプの契約を守っているか
- I: 不要な依存を強制していないか
- D: 抽象に依存しているか

## 4. リファクタリング計画
改善項目と実行順序を提示し、ユーザーの承認後に実行。

## 5. 実行と検証
- 各ステップでテストが通過することを確認
- 振る舞いが変わっていないことを検証

結果を Before/After の比較で報告してください。
