# 実装計画コマンド

以下の手順で `$1` の実装計画を作成してください：

## 1. 要件分析
- 機能要件と非機能要件を整理
- 不明点があれば質問として列挙

## 2. 現状調査
- 関連する既存コードを特定（`@` 参照で確認）
- 使用中の技術スタック・パターンを確認

## 3. 影響範囲の特定
変更が影響するファイルを一覧化:
```
- `path/to/file` — 変更内容
```

## 4. タスク分解
実装をステップに分解:
```
1. [ ] タスク名 (S/M/L) — 説明
   - 対象: `ファイルパス`
   - 依存: なし
```

サイズ目安: S=数行, M=50行程度, L=100行以上

## 5. リスクと完了条件
- リスク → 対策
- 完了チェックリスト（テスト通過、型エラーなし等）

## 6. 承認確認
計画を提示後、ユーザーの承認を待ってから実装を開始してください。
