# テスト生成コマンド

以下の手順で `$1` のテストを作成してください：

## 1. 対象の分析
- `@` 参照でテスト対象のコードを確認
- 関数・コンポーネントの入出力を把握
- 依存関係を確認

## 2. テストケースの設計

### 正常系（ハッピーパス）
- 期待通りの入力で期待通りの出力が得られるか

### 異常系
- 不正な入力に対するエラーハンドリング
- 境界値の処理

### エッジケース
- 空配列、空文字列、null/undefined
- 最大値、最小値

## 3. テストの実装

プロジェクトのテストフレームワークに合わせて AAA パターンで記述:
- Arrange: テストデータの準備
- Act: テスト対象の実行
- Assert: 結果の検証

### 規約
- テスト名は日本語で「何をしたら何が起きるか」を記述
- 1テスト1アサーション（関連するアサーションはグループ化可）
- テスト間で状態を共有しない
- モックは最小限に

## 4. テストの実行

プロジェクトの技術スタックに応じたテスト実行コマンドを使用してください。

## 5. 結果の報告

```markdown
## テスト作成結果
- テストファイル: `path/to/test.ts`
- テストケース数: N
- カバレッジ: N%
```
