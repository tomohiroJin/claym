# コーディングスタイル規約

以下はプロジェクトの技術スタックに応じて適用してください。

## TypeScript（該当プロジェクトの場合）

### 型の使用
- `any` 型は使用禁止 → `unknown` + 型ガードを使用
- `null` より `undefined` を優先（外部 API の境界を除く）
- 型推論が十分な場合は明示的な型注釈を省略可

### 関数
- 1関数1責務（単一責任の原則）
- 30行以内を目安（超える場合は分割を検討）
- 早期リターンでネストを浅く保つ
- パラメータは3個以内（超える場合はオブジェクトにまとめる）

### 変数
- `const` を優先、`let` は必要な場合のみ、`var` は禁止
- マジックナンバーは名前付き定数に置換
- ブール変数は `is`, `has`, `can`, `should` 接頭辞を使用

### インポート
- 相対パスのインポートは `../` の階層を2つまでに制限
- 名前付きエクスポートを優先（デフォルトエクスポートは避ける）
- 未使用のインポートは削除

## React（該当プロジェクトの場合）

### コンポーネント
- 関数コンポーネントのみ使用
- Props は型定義を必ず行う
- コンポーネントは200行以内を目安
- 状態管理のロジックはカスタムフックに分離

### JSX
- 条件付きレンダリングは早期リターンまたは三項演算子
- リストの `key` には安定した一意値を使用（index は非推奨）

## 命名規則

| 対象 | 規則 | 例 |
|------|------|-----|
| 変数・関数 | camelCase | `getUserName` |
| クラス・型 | PascalCase | `UserProfile` |
| 定数 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| ファイル名 | kebab-case | `user-profile.ts` |
| コンポーネント | PascalCase | `UserProfile.tsx` |
| テスト | *.test.ts(x) | `user-profile.test.ts` |

## コメント

- 日本語で記述
- 「なぜ」を説明するコメントを優先（「何を」はコードで表現）
- TODO には日付と担当者を記載: `// TODO(2025-01-01): 説明`
