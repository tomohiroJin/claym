# ビルドエラー修正コマンド

ビルドエラーや型エラーを解析し、修正します。

## 使い方

```
/build-fix <エラーの説明やコンテキスト（省略可）>
```

## プロンプト

以下の手順でビルドエラーを解決してください：

### 1. エラーの収集

プロジェクトの技術スタックを確認し、適切なビルドコマンドを実行してエラーを収集します：

```bash
# プロジェクトの技術スタックに応じたコマンドを選択
# Node.js/TypeScript: npx tsc --noEmit && npm run build
# Python: mypy src/ && python -m pytest
# Go: go build ./...
# Rust: cargo build
```

ユーザーが追加情報 `$1` を提供している場合、それも考慮してください。

### 2. エラーの分類

検出されたエラーを分類します：

| 分類 | 例 |
|------|-----|
| 型・コンパイルエラー | 型不一致、未定義シンボル |
| インポート・依存エラー | モジュール未解決、パス間違い |
| 設定エラー | ビルド設定・コンパイラ設定の問題 |
| 依存関係エラー | パッケージ未インストール、バージョン不一致 |

### 3. 原因の特定

各エラーについて：

- エラーメッセージの意味を説明
- 該当ファイルの関連コードを確認
- 根本原因を特定（1つのエラーが複数の派生エラーを生むケースに注意）

### 4. 修正の実行

- **連鎖エラーを優先**: 1つ直すと複数解決するエラーから対応
- **最小限の変更**: 既存の動作を壊さない修正を選択
- 修正前後のコードを明示

### 5. 修正後の確認

プロジェクトの技術スタックに応じたチェックコマンドを実行します：

- 型チェック / コンパイル
- ビルド
- テスト実行

### 6. 結果レポート

```markdown
## ビルドエラー修正結果

### 修正したエラー
1. **エラー内容** — `ファイル:行番号`
   - 原因: [原因]
   - 修正: [修正内容]

### 確認結果
- [ ] 型チェック通過
- [ ] ビルド成功
- [ ] テスト通過
```
