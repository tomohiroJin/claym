# 実装計画コマンド

要件を分析し、実装計画を作成します。

## 使い方

```
/plan <実装したい機能や変更の説明>
```

## プロンプト

以下の手順で実装計画を作成してください：

### 1. 要件分析

ユーザーの要求 `$1` を分析します：

- **機能要件**: 何を実現するか
- **非機能要件**: パフォーマンス、セキュリティ等の制約
- **不明点**: 曖昧な部分があれば質問として列挙

### 2. 現状調査

プロジェクトの現状を調査します：

- 関連する既存コードを特定
- 使用中の技術スタック・パターンを確認
- 類似の実装があれば参考にする

### 3. 影響範囲の特定

変更が影響するファイルを一覧化します：

```markdown
## 影響範囲
- `src/path/to/file.ts` — 変更内容の概要
- `src/path/to/test.ts` — テストの追加・修正
```

### 4. タスク分解

実装をステップに分解します：

```markdown
## タスク一覧
1. [ ] **タスク名** (S/M/L) — 説明
   - 対象ファイル: `パス`
   - 依存: なし
2. [ ] **タスク名** (S/M/L) — 説明
   - 対象ファイル: `パス`
   - 依存: タスク1
```

サイズの目安:
- **S**: 1ファイル、数行の変更
- **M**: 2-3ファイル、50行程度の変更
- **L**: 4ファイル以上、100行以上の変更

### 5. リスクと完了条件

```markdown
## リスク
- [リスク] → [対策]

## 完了チェックリスト
- [ ] すべてのタスクが完了
- [ ] テストが追加・通過
- [ ] 型エラーなし
- [ ] 既存テストが通過
```

### 6. 承認の確認

計画を提示した後、ユーザーに承認を求めてください。
承認後に実装を開始します。
